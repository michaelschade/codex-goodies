# Prompt Writing

This skill matters when the work is genuinely about prompt quality: writing a strong prompt from scratch, repairing a prompt after a specific failure mode, or deciding whether a "prompt problem" is actually a surface-design problem.

## Why You Might Care

- Use it when you need a system prompt, developer prompt, reusable task prompt, or tool-using prompt that fits a real runtime instead of the authoring session.
- Use it when an existing prompt mostly works and you want the smallest high-leverage edits rather than a full rewrite.
- Use it when you need a clean handoff between prompt surgery and the broader Codex-surface decisions that may belong in `$codex-docs`.

If you want the operational rules, checklists, and scaffolds, jump to [`SKILL.md`](SKILL.md).
