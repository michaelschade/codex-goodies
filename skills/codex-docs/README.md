# Codex Docs

This skill matters when the real question is how Codex itself should be set up, structured, or used. It helps with Codex surfaces, docs-guided workflow choices, and the durable decisions behind a clean workflow.

## Why You Might Care

- Use it when a workflow keeps getting heavier prompts instead of a cleaner hook, skill, subagent, or agent design.
- Use it when you need to decide whether something belongs in `AGENTS.md`, a skill, config, automation, `codex exec`, the SDK, or some other Codex surface.
- Use it when the recommendation should be grounded in Codex docs and current local setup rather than in generic prompt advice.

If you already know you need the full operating rules, examples, and references, jump to [`SKILL.md`](SKILL.md).
